# 룰렛 투게더 사용 가이드

## 방 생성 및 입장 기능

### 1. 방 생성하기 (방장)

#### 단계별 프로세스

1. **메인 화면에서 "방 만들기" 버튼 클릭**
   - Stepper 다이얼로그가 열립니다

2. **Step 1: 닉네임 입력 (선택)**
   - 닉네임을 입력하거나 건너뛸 수 있습니다
   - 미입력 시 서버에서 자동으로 닉네임 생성 ("생성자")
   - 최대 20자까지 입력 가능

3. **Step 2: 룰렛 설정**
   - **당첨자 수**: 1~100명 사이의 숫자 입력
   - **당첨 감정**:
     - 긍정적 (POSITIVE): 선물 추첨, 청소 면제 등
     - 부정적 (NEGATIVE): 설거지, 돈내기 등

4. **Step 3: 확인 및 생성**
   - 입력한 정보 확인
   - "방 만들기" 버튼 클릭
   - 자동으로 방 화면으로 이동

#### 방장 화면 기능

- **참가자 목록 확인**
  - 실시간으로 참가자 입장 확인
  - 각 참가자의 준비 상태 표시
  - 준비 완료/대기 중 뱃지로 구분

- **링크 공유**
  - "참가자 링크 공유" 버튼으로 링크 복사
  - 모바일에서는 네이티브 공유 기능 사용 가능

- **룰렛 돌리기**
  - 모든 참가자가 준비 완료해야 활성화
  - 버튼 클릭 시 룰렛 시작

### 2. 방 참가하기 (참가자)

#### 참가 프로세스

1. **공유받은 링크 클릭**
   - 링크 형식: `/room/{roomId}?role=participant`

2. **닉네임 입력 (선택)**
   - 다이얼로그에서 닉네임 입력
   - 미입력 시 서버에서 자동 생성 ("참가자 N")
   - 최대 20자까지 입력 가능

3. **방 입장 완료**
   - 자동으로 방 대기실로 이동

#### 참가자 화면 기능

- **닉네임 변경**
  - 연필 아이콘 클릭하여 닉네임 수정
  - 변경 시 방장에게 실시간 반영

- **준비 상태 관리**
  - "준비 완료" 버튼으로 준비 상태 토글
  - 준비 완료 시 방장에게 알림
  - 준비 취소도 가능

- **룰렛 설정 확인**
  - 방장이 설정한 당첨자 수와 당첨 감정 확인

### 3. WebSocket 연결 상태

#### 연결 중

- 로딩 스피너와 "서버에 연결하는 중..." 메시지 표시

#### 연결 실패

- 자동 재연결 시도 (최대 5회)
- 재연결 간격: 1초

#### 방 입장 거부

- 이유별 에러 메시지 표시:
  - `OWNER_ALREADY_EXISTS`: 이미 방장이 존재함
  - `INVALID_REQUEST`: 잘못된 요청
  - `INVALID_RID`: 유효하지 않은 RID

## 기술 구현 세부사항

### 상태 관리 (Zustand)

```typescript
// 전역 상태 구조
{
  roomId: string | null;           // 방 ID
  role: 'owner' | 'participant';   // 역할
  isOwner: boolean;                // 방장 여부
  myNickname: string;              // 내 닉네임
  myRid: string;                   // 내 RID
  config: {                        // 방 설정
    winnersCount: number;
    winSentiment: 'POSITIVE' | 'NEGATIVE';
  };
  participants: Participant[];     // 참가자 목록 (방장만)
  myReady: boolean;                // 내 준비 상태 (참가자만)
}
```

### WebSocket 이벤트

#### Client → Server

- `room:join`: 방 입장 요청
- `participant:ready:toggle`: 준비 상태 토글
- `participant:nickname:change`: 닉네임 변경
- `spin:request`: 룰렛 스핀 요청 (방장만)

#### Server → Client

- `room:joined`: 방 입장 완료
- `room:join:rejected`: 방 입장 거부
- `room:config`: 방 설정 정보
- `room:participants`: 참가자 목록 (방장에게만)
- `nickname:changed`: 닉네임 변경 확인
- `spin:resolved`: 스핀 시작
- `spin:outcome`: 개인 결과
- `spin:result`: 전체 결과

### 컴포넌트 구조

```
RoomPage (widgets)
├── useSocket() - WebSocket 연결
├── useRoomEvents() - 이벤트 처리
└── RoomWaiting (features)
    ├── OwnerView (방장)
    │   ├── 참가자 목록
    │   ├── 준비 상태 확인
    │   └── 룰렛 돌리기 버튼
    └── ParticipantView (참가자)
        ├── 닉네임 변경
        ├── 준비 상태 토글
        └── 대기 화면
```

## 환경 설정

### 필수 환경 변수

`.env.local` 파일 생성:

```env
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_WS_URL=http://localhost:3001
NEXT_PUBLIC_FRONTEND_URL=http://localhost:3000
```

### 백엔드 서버 실행

방 생성 및 입장 기능을 테스트하려면 백엔드 서버가 실행 중이어야 합니다:

```bash
# 백엔드 서버 실행 (별도 터미널)
cd ../roulette-together-be
npm run start:dev
```

### 프론트엔드 실행

```bash
# 의존성 설치
pnpm install

# 개발 서버 실행
pnpm dev
```

브라우저에서 `http://localhost:3000` 접속

## 테스트 시나리오

### 시나리오 1: 기본 방 생성 및 참가

1. 브라우저 A에서 방 생성
2. 닉네임 "방장" 입력
3. 당첨자 수 3명, 긍정적 감정 선택
4. 방 생성 완료 후 참가자 링크 복사
5. 브라우저 B에서 링크로 접속
6. 닉네임 "참가자1" 입력
7. 준비 완료 버튼 클릭
8. 브라우저 A에서 참가자 목록 확인

### 시나리오 2: 닉네임 자동 생성

1. 방 생성 시 닉네임 입력하지 않음
2. 자동으로 "생성자" 닉네임 부여됨
3. 참가자도 닉네임 입력하지 않음
4. 자동으로 "참가자 1", "참가자 2" 등으로 부여됨

### 시나리오 3: 닉네임 변경

1. 참가자로 방 입장
2. 연필 아이콘 클릭
3. 새 닉네임 입력 후 저장
4. 방장 화면에서 실시간으로 변경된 닉네임 확인

### 시나리오 4: 준비 상태 관리

1. 참가자 3명 입장
2. 각각 준비 완료 버튼 클릭
3. 방장 화면에서 "3/3명 준비 완료" 확인
4. 룰렛 돌리기 버튼 활성화 확인
5. 한 명이 준비 취소
6. 룰렛 돌리기 버튼 비활성화 확인
